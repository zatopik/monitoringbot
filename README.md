# 🤖 Nix-Admin AI Bot

Локальный Telegram-бот для администрирования Linux-серверов с интегрированным искусственным интеллектом (Ollama). Проект реализован на **Go** с использованием **Nix Flakes** для обеспечения 100% воспроизводимости окружения.

## 🌟 Основные возможности

- **Интеллектуальный мониторинг**: Сбор метрик CPU/RAM и их анализ нейросетью (`qwen2.5-coder` или `llama3`).
- **Управление сервисами**: Проверка статуса, чтение логов (`journalctl`) и безопасный перезапуск через `/restart`.
- **Контроль инфраструктуры**: Мониторинг Docker-контейнеров, открытых портов и сетевой активности.
- **Умные уведомления (Alerts)**: Фоновый мониторинг ресурсов с автоматическим оповещением и советами от ИИ при перегрузках.
- **Безопасность**: Доступ только по белому списку Telegram ID и управление секретами через переменные окружения.
- **Nix-интеграция**: Декларативное описание всех зависимостей (Go, Ollama, системные утилиты).

## 🛠 Технологический стек

- **Language:** Go 1.22+
- **AI Engine:** Ollama (Local LLM)
- **Environment:** Nix Flakes
- **OS:** NixOS / Any Linux
- **Libraries:** 
  - `go-telegram-bot-api` — работа с мессенджером.
  - `gopsutil` — сбор системных метрик.
  - `ollama/api` — взаимодействие с локальной нейросетью.

## ⌨️ Список команд бота

Команды разделены на логические блоки для удобства администрирования. Каждая команда сопровождается анализом от локальной нейросети.

### 📊 Мониторинг ресурсов

| Команда | Описание | Что делает ИИ |
|:---|:---|:---|
| `/status` | Состояние CPU и RAM | Анализирует текущую нагрузку и дает прогноз |
| `/top` | Топ-15 процессов | Находит процессы, потребляющие больше всего ресурсов |
| `/net` | Сетевая активность | Проверяет RX/TX трафик на предмет аномалий |
| `/ports` | Открытые порты | Подсказывает, какие порты стандартные, а какие подозрительные |

### 🐳 Инфраструктура и Сервисы

| Команда | Описание | Что делает ИИ |
|:---|:---|:---|
| `/docker` | Статус контейнеров | Оценивает состояние запущенных Docker-контейнеров |
| `/nginx` | Веб-сервер | Проверяет работоспособность и дает краткий отчет |
| `/systemd` | Системные службы | Ищет упавшие (`failed`) юниты во всей системе |

### 🛠 Управление и Отладка

| Команда | Описание | Что делает ИИ |
|:---|:---|:---|
| `/logs <svc>` | Чтение логов | Анализирует последние 15 строк `journalctl` на ошибки |
| `/restart <svc>` | Перезапуск службы | Выполняет `sudo restart` и подтверждает успех |
| `/cat <path>` | Чтение файла | Проверяет содержимое конфига на наличие опечаток |
| `/clear` | Очистка памяти | Стирает контекст текущего диалога с нейросетью |

### 💬 Обычное общение
Если отправить боту обычное текстовое сообщение, он ответит в режиме **ассистента-сисадмина**, сохраняя контекст последних 10-15 реплик.
